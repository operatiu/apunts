    RA1- Serveis de configuració dinàmica (DHCP)¶

    El servidor DHCP asigna automàticament una IP a cada client que es connecta, evitant conflictes i facilitant l'administració de la xarxa.

    Instal·lació i configuració de DHCP a GNU/Linux¶

    Missatges DHCP (DORA: Discover, Offer, Request, Acknowledge [ACK / NACK] ).

    • Exemple: subnet 172.16.X.0/24 amb IPs del 40 al 80.  
    • Activitat: Instal·lació i configuració isc-dhcp-server i reserva d’una IP per MAC.  
    • Comprovació amb ip a, journalctl, ifdown enp2s0 / ifup enp2s0, dhclient.

    IP fixa al servidor

    Arxiu clàssic de networking

    sudo nano /etc/network/interfaces
    allow-hotplug enp2s0
auto enp2s0
iface enp2s0 inet static
    address 172.16.0.1      
    # o amb màscara i ens estalvien netmask:  address 172.16.0.1/24
    netmask 255.255.255.0

    Un cop desat l’arxiu

    sudo service networking restart

    o bé si volem una interfície determinada per defecte

    sudo ifdown enp2s0  2>/dev/null
sudo ifup enp2s0

    Si configurem amb Netplan: Compte amb el sagnat (doble espai)

    sudo nano /etc/netplan/01-netcfg.yaml
    network:
  version: 2
  ethernets:
    enp2s0:
      addresses: [172.16.0.1/24]
      # dhcp4: true

    Un cop desat l’arxiu

    sudo netplan apply

    Comprovació d’IP

    ip -br a
    o 
    ip link show

    isc-dhcp-server¶

    Instal·lar el servidor DHCP¶

    sudo apt update
sudo apt install isc-dhcp-server -y

    Normalment apareixen errors quan intentem aixecar el servei després de la instal·lació.

    Els arreglarem de la següent forma.

    Configurar la interfície a on escoltar

    sudo nano /etc/default/isc-dhcp-server
    INTERFACESv4="enp2s0"
INTERFACESv6=""

    Configurar el rang d’adreces DHCP 

    sudo nano /etc/dhcp/dhcpd.conf

    option domain-name "lab.local";
option domain-name-servers 8.8.8.8, 1.1.1.1;

default-lease-time 600;
max-lease-time 7200;

    Si volem podem fixar el servidor com autoritatiu descomentant la línia

    # authoritative

    Al final de l’arxiu afegirem

    subnet 172.16.0.0 netmask 255.255.255.0 {
    range 172.16.0.80 172.16.0.120;
    option routers 172.16.0.1;
    option broadcast-address 172.16.0.255;
}

    Desarem l’arxiu i reiniciarem el servidor 

    sudo systemctl restart isc-dhcp-server
sudo service isc-dhcp-server status

    Si apareixen errors podem mirar d’acotar-los

    Veure estat i últims errors 

    sudo systemctl status isc-dhcp-server -l

    Validar sintaxi 

    sudo dhcpd -t -cf /etc/dhcp/dhcpd.conf

    Assegurar que la interfície està activa 

    ip a show enp2s0

    Comprovar logs 

    journalctl -xeu isc-dhcp-server 

    Forçar execució i veure que no torna cap error  

    sudo dhcpd -t -cf /etc/dhcp/dhcpd.conf

    Buidar les concessions per si hi ha fitxers corruptes. 

    sudo rm -f /var/lib/dhcp/dhcpd.leases
sudo touch /var/lib/dhcp/dhcpd.leases
sudo systemctl restart isc-dhcp-server

    PROVES

    Concessions atorgades 

    sudo tail -n 50 /var/lib/dhcp/dhcpd.leases

    Capturar DORA (DHCP) a la interfície interna 

    sudo tcpdump -i enp2s0 -n -vv port 67 or port 68

    Ampliació: Reserva IP per MAC del client.

    host clientfix {
  hardware ethernet 08:00:224:aa:bb:cc;
  fixed-address 172.16.0.200;
}

    CLIENT

    El client el configurarem com DHCP 

    sudo nano /etc/network/interfaces

    auto enp1s0
iface enp1s0 inet dhcp

    Comprovacions

    sudo ifdown enp1s0 2>/dev/null
sudo ifup enp1s0

    O el podem forçar amb dhclient si tenim el paquet instal·lat

    sudo dhclient -r enp1s0
sudo dhclient -v enp1s0

    Fer servir el server com enrutador¶

    Al servidor

    IP forwarding + NAT 

    sudo nano -l /etc/sysctl.conf

    Comprovem que la següent línia (28) estigui descomentada 

    net.ipv4.ip_forward=1

    A partir de Debian 12 i 13 Crear arxiu personalitzat: sudo nano /etc/sysctl.d/99-ipforward.conf

    contingut:  

    net.ipv4.ip_forward=1
    # net.ipv6.conf.all.forwarding=1 # opcional

    Apliquem canvis des del terminal un cop desat 

    sudo sysctl --system

    El verifiquem: cat /proc/sys/net/ipv4/ip_forward  Si torna 1, és correcte

    NAT amb iptables 

    enp1s0 (WAN/NAT) i enp3s0 (LAN interna) Permetre reenviament

    Si cal instal·lem: sudo apt install iptables

    sudo iptables -A FORWARD -i enp3s0 -o enp1s0 -j ACCEPT
sudo iptables -A FORWARD -i enp1s0 -o enp3s0 -m state --state ESTABLISHED,RELATED -j ACCEPT

    NAT (masquerade) Necessari perque els clients sortin a Internet per la interfície enp1s0 del server 

    sudo iptables -t nat -A POSTROUTING -o enp1s0 -j MASQUERADE
    O indicant la xarxa (trànsit de la LAN) afegint  -s 
    sudo iptables -t nat -A POSTROUTING -s 172.15.0.0/24 -o enp1s0 -j MASQUERADE

    Si volem desar les regles de forma persistent

    sudo apt install -y iptables-persistent netfilter-persistent
sudo netfilter-persistent save
    Les regles es guardaran a /etc/iptables/rules.v4

    Comprovació de regles i forwarding

    Comprovar forwarding actiu 

    cat /proc/sys/net/ipv4/ip_forward

    Comprovar iptables 

    sudo iptables -t nat -S
sudo iptables -S

    Eliminar les regles en execució (temporalment fins al reinici)

    Netejar totes les cadenes i regles: 

    sudo iptables -F
sudo iptables -t nat -F
sudo iptables -t mangle -F
sudo iptables -X

    Això buida totes les regles, però en reiniciar la màquina, tornaran les desades.

    Esborrar les regles persistents  Si vols que no es tornin a aplicar al reinici:

    Buidar regles actuals 

    sudo iptables -F
sudo iptables -t nat -F
sudo iptables -t mangle -F
sudo iptables -X
    Guardar la configuració buida:  sudo netfilter-persistent save

    La propera vegada que arrenquem, tindrem la taula neta.

    Eliminar completament el sistema de regles persistents

    Per treure-ho del tot: 

    sudo apt remove --purge iptables-persistent netfilter-persistent -y

    Eliminar fitxers de configuració residuals que puguin haver quedat

    sudo rm -f /etc/iptables/rules.v4
sudo rm -f /etc/iptables/rules.v6

    Per Debian 12, 13 o + farem servir NFTABLES

    Activem el servei: 

    sudo systemctl enable --now nftables

    Editem la configuració: 

    sudo nano /etc/nftables.conf

    Afegim la configuració d’interfícies 

    table ip nat {
    chain postrouting {
        type nat hook postrouting priority 100;
        oif "enp1s0" masquerade
    }
}

table ip filter {
    chain forward {
        type filter hook forward priority 0;
        iif "enp3s0" oif "enp1s0" accept
        iif "enp1s0" oif "enp3s0" ct state related,established accept
    }
}

    Un cop desat l’arxiu, el carreguem i comprovem el funcionament 

    sudo nft -f /etc/nftables.conf
ping 8.8.8.8
ping www.debian.org

    KEA-DHCP4-SERVER¶

    CONFIGURACIÓ i INSTAL·LACIÓ¶

    Modifiquem els arxius del SSOO¶
    sudo nano /etc/hostname
    deb13srv0

    Nota: on 0 l’has de substituir pel teu número X de xarxa (Ex. 172.16.5.0)

    sudo nano /etc/hosts
    127.0.0.1       localhost
172.15.0.1      deb13srv0.2smx.local deb13srv0

    Els clients es poden afegir manualment per fer proves

    • 172.15.0.10     deb13cli1.2smx.local deb13cli1
    • 172.15.0.11     wincli1.2smx.local wincli1

    Des del terminal el fem persistent

    hostnamectl
    o 
    sudo hostnamectl set-hostname deb13srv0

    Podem reiniciar màquina o executar: exec bash

    Instal·lem KEA¶

    sudo apt install kea-dhcp4-server

    Fem còpia de l’arxiu de configuració i l’editem o crearem un de nou

    /etc/kea/kea-dhcp4.conf

    Amb el contingut mínim:

    {
    "Dhcp4": {
        "interfaces-config": {
            "interfaces": [ "enp3s0" ]
        },
        "subnet4": [{
            "subnet": "172.15.0.0/24",
            "id": 1,
            "pools": [
                { "pool": "172.15.0.50-172.15.0.80" }
            ],
            "reservations": [
                { "hw-address": "52:54:00:69:93:e1", "ip-address": "172.15.0.12" } # Debian13_2SMX
            ],
            "option-data": [
                { "name": "domain-name",
                  "data": "2smx.local"
                },
                { "name": "domain-name-servers", 
                  "data": "172.15.0.1, 8.8.8.8, 8.8.4.4" 
                },
                { "name": "routers", 
                  "data": "172.15.0.1" }
            ]
        }],
"loggers": [
      { "name": "kea-dhcp4",
        "output_options": [
          { "output": "/var/log/kea/kea-dhcp4.log"
          }
        ],
        "severity": "INFO",
        "debuglevel": 0
      }
    ]
    }
}

    Reiniciem el servei i el verifiquem 

    sudo systemctl restart kea-dhcp4-server
sudo systemctl status kea-dhcp4-server -l

    Les concessions es guardaran a: 

    /var/lib/kea/kea-dhcp4.leases

    Secció Significat
    "interfaces-config" Indica en quina interfície escoltarà el servidor DHCP (enp3s0 a la xarxa interna).
    "lease-database" On es guarden les concessions. Aquí s’usa un fitxer temporal (memfile) en /var/lib/kea/dhcp4.leases.
    "valid-lifetime" Temps en segons que dura una concessió (3600s = 1 hora).
    "renew-timer" i "rebind-timer" Temps perquè el client demani renovació de la IP.
    "option-data" Paràmetres globals: domini, DNS, i router (gateway).
    "subnet4" Defineix la subxarxa, el rang d’adreces IP i opcions específiques per a ella.
    "loggers" Registra missatges del servei a /var/log/kea-dhcp4.log.
