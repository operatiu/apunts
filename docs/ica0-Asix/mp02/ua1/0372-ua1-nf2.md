# UA 1 - AEA2: El model Entitat/Relació

## AEA2 - Model E/R (Entitat interrelació)

## Model Entitat - Relació

També anomenat Entitat Interrelació o E/R

El model E/R que començarem a fer servir és el proposat per Peter Chen a principis dels anys '70 i que més tard donarà lloc al model E/R estès proposat per Teorey Yang (a finals dels '80).

### Objectius

- Conèixer les etapes que integren el procés del disseny d’una base de dades.
- Conèixer les estructures del model ER.
- Saber fer el disseny conceptual de les dades d’un sistema d’informació mitjançant el model ER.
- Saber fer el disseny lògic d’una base de dades relacional partint d’un disseny conceptual expressat amb el model ER.

### Introducció

El disseny d’una base de dades consisteix a definir l’estructura de les dades que ha de tenir la base de dades d’un sistema d’informació (SI) determinat.

En el cas relacional, aquesta estructura serà un conjunt d’esquemes de relació amb els seus atributs, dominis d’atributs, claus primàries, claus foranes, etc

### Etapes del disseny de bases de dades

- Etapa del disseny conceptual
- - El resultat de l’etapa del disseny conceptual s’expressa mitjançant algun model de dades d’alt nivell. Un dels més emprats és el model entitat-interrelació
    - (entity-relationship), que abreviarem amb la sigla ER.
- Etapa del disseny lògic
- - Aquesta etapa parteix del fet que ja s’ha resolt la problemàtica de l’estructuració de la informació a un nivell conceptual i ens permet concentrar-nos en les qüestions tecnològiques relacionades amb el model de base de dades.
    - Per exemple, si es tracta d’un SGBD relacional, aquesta etapa obtindrà un conjunt de relacions amb els seus atributs, claus primàries i claus foranes.
- Etapa del disseny físic
- - El resultat de l’etapa del disseny físic se situa al món de les representacions, igual que el resultat de l’etapa del disseny lògic. La diferència respecte a l’etapa anterior és que ara es tenen en compte aspectes de caràcter més físic del món de les representacions. En aquesta etapa es transforma l’estructura obtinguda a l’etapa del disseny lògic amb l’objectiu d’aconseguir una major eficiència i, a més, es completa amb aspectes d’implementació física que dependran de l’SGBD.

Conjunts d’eines lògiques per descriure les dades, les seves interrelacions, el seu significat i les restriccions a aplicar per tal de garantir-ne la coherència.

### Disseny conceptual. El model ER

Entity-relationship (Entitat-Interrelació)

Aplicable al disseny de qualsevol tipus de base de dades (relacional, jeràrquica, etc.)

Peter Chen (1976)

#### Entitats i Atributs

<img src="https://res.cloudinary.com/dbuv9r3p5/image/upload/v1767873750/pasted_image_0_tbagnn.png" width="240">

Per entitat entenem un objecte del món real que podem distingir de la resta d’objectes i del qual ens interessen algunes propietats.

- Instàncies o ocurrències concretes.
- Tipus o classes d’entitats.

Les propietats dels objectes que ens interessen s’anomenen atributs.

- Univaluat
- Clau
- Clau Candidata
- Clau Primària
